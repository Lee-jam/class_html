### Iframe
------------
```
+ iframe은 웹페이지 내에 웹페이지를 표시하는데 사용함.
+ iframe은 인라인 프레임을 지정함. 인라인프레임은 다른 문서를 포함하는데 사용함.
+ 스크린 리더가 iframe의 내용을 읽는데 사용하기 때문에 항상 title 속성을 포함하는 것이 좋음.
+ iframe은 height와 width로 크기를 지정할 수 있음. (픽셀 단위)
+ iframe은 기본적으로 테두리를 가지고 있기 때문에 border속성을 이용하여 삭제할 수 있음.
+ iframe은 링크의 대상 프레임으로 사용할 수 있음.
 - traget링크의 속성을 참조해야함. a의 target = iframe의 name
 
```

### File Path
------------
```
+ 파일 경로는 웹사이트의 폴더 구조에서 파일의 위치를 설명함.
+ ex) img src = "img.jpg" 파일은 현재 페이지와 같은 폴더에 있음.
+ img src = "picture/img.jpg 파일은 현재 폴더의 picture폴더 내에 있음.
+ img src="/images/picture.jpg"> 파일은 현재 웹의 루트에 있는 images 폴더에 있음
+ img src="../picture.jpg" 파일은 현재 폴더에서 한 단계 위의 폴더에 있음
+ 파일 경로는 웹 페이지, 이미지, 스타일시트, 자바스크립트와 같은 외부 파일에 연결할 때 사용됨.
+ 절대 파일 경로는 파일의 전체 URL을 의미함.
```

### Head Elements
------------
```
+ 헤드 요소는 메타 데이터(데이터에 대한 데이터)에 대한 컨테이너이고 <html>태그와 <body>태그 사이에 위치함.
+ HTML 메타데이터는 HTML문서에 대한 데이터이며 표시되지 않음.
+ 메타데이터는 일반적으로 문서 제목, 문자 집합, 스타일, 스크립트 및 기타 메타 정보를 정의함.
+ <title>요소는 문서의 제목을 정의하고 페이지 제목의 내용은 검색엔진 최적화(SEO)에 매우 중요함.
+ <title>요소는 브라우저 도구 모음에서 제목을 정의하고 즐겨찾기에 추가될 때 페이지의 제목을 제공, 검색 엔진 결과에 페이지 제목을 표시함.
+ <style>요소는 스타일 정보를 정의하는데 사용함.
+ <link>요소는 현재 문서 및 외부 자원 사이의 관계를 정의함.
+ <meta>요소는 일반적으로 문자 집합, 페이지 설명, 키워드, 문서의 저자, 뷰포트 설정을 지정하는 데 사용됨.
 - <meta charset="UTF-8"> : 사용된 문자 집합을 정의
 - <meta name="keywords" content="HTML, CSS, JavaScript"> : 검색 엔진에 대한 키워드 정의
 - <meta name="description" content="Free Web tutorials"> : 웹 페이지에 대한 설명을 정의
 - <meta name="author" content="John Doe"> : 페이지 작성자 정의
 - <meta http-equiv="refresh" content="30"> : 30초마다 문서 새로 ​​고침
 - <meta name="viewport" content="width=device-width, initial-scale=1.0"> : 웹사이트가 모든 기기에서 잘 보이도록 표시 영역 설정
+ 뷰포트는 웹 페이지에서 사용자가 볼 수 있는 영역이며 기기에 따라 다름.
+ 뷰포트는 페이지의 크기와 배율을 제어하는 방법에 대한 브라우저 지침을 제공함.
+ width=device-width부분 장치의 화면 폭을 따라 페이지의 폭을 설정함.
+ 이 initial-scale=1.0부분은 브라우저에서 페이지를 처음 로드할 때 초기 확대/축소 수준을 설정함.
+ <base>요소는 페이지의 모든 상대 URL의 기본 URL 및 또는 대상을 지정함.
+ HREF 또는 대상 속성 존재 또는 둘 모두를 가져야 함.
```

### Layout Elements and Techniques
------------
```
+ <header> - 문서 또는 섹션의 헤더를 정의함.
+ <nav> - 탐색 링크 세트를 정의함.
+ <section> - 문서의 섹션을 정의힘.
+ <article> - 독립적이고 독립적인 콘텐츠를 정의함.
+ <aside> - 콘텐츠 이외의 콘텐츠 정의(사이드바 등)
+ <footer> - 문서 또는 섹션의 바닥글을 정의함.
+ <details> - 사용자가 필요에 따라 열고 닫을 수 있는 추가 세부 정보 정의
+ <summary>- <details>요소 의 제목을 정의함.
+ 레이아웃 기법에서 네가지 기술이 있음. 각 기술에는 장단점이 있음.
 - CSS 프레임워크
  - 레이아웃을 빠르게 만들고 싶다면 W3.CSS 또는 Bootstrap 과 같은 CSS 프레임워크를 사용할 수 있음.
  - CSS float 속성을 사용하여 전체 웹 레이아웃을 수행하는 것이 일반적임.
 - CSS 부동 속성
  - float 및 clear 속성이 어떻게 작동하는지 기억하기만 하면 됨.
  - ※주의=부동요소는 문서 흐름에 연결되어 유연성에 해를 끼칠 수 있음.
 - CSS 플렉스박스
  - flexbox를 사용하면 페이지 레이아웃이 다양한 화면 크기와 다양한 디스플레이 장치를 수용해야 할 때 요소가 예측 가능하게 작동함.
 - CSS 그리드 레이아웃
  - 그리드 모듈은 행과 열이 있는 그리드 기반 레이아웃 시스템을 제공하므로 부동 소수점과 위치지정을 사용하지 않고도 웹 페이지를 더 쉽게 디자인할 수 있음.
```
